## Optional Stretch Goals

Optional stretch-goal material is included under `bonus/`. These files do not
replace the required Bash, systemd, Nginx, UFW, validation, and reboot-survival
flow.

- `bonus/ansible/`: local Ansible equivalent of the provisioning flow.
- `bonus/vm-snapshot-and-template/`: local snapshot, restore, and template notes.
- `bonus/monitoring/`: service, HTTP, port, firewall, timer, and log checks.
- `bonus/docker/`: optional container build for the Python backend.
- `bonus/rollback/`: dry-run-first rollback script with explicit confirmation.

Bonus evidence should be captured only after the required milestones are
complete.
